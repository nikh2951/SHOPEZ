Thank you for your interest in contributing to ShopEZ.

Quick guidelines:

- Fork the repository and create a feature branch: `feature/short-description`.
- Keep PRs small and focused; include screenshots for UI changes.
- Write clear commit messages and a concise PR description explaining the change and motivation.
- For new features or bug fixes, include steps to reproduce and any relevant logs.
- Run existing linters/tests before opening a PR.

Code of conduct: Be respectful and inclusive when interacting in issues and PRs.

Maintainers will review and respond to PRs as time permits.
