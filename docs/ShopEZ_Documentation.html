Overview

ShopEZ is a modern e-commerce demo application consisting of a React + Vite frontend and an Express + Node backend. The backend uses mongoose for data modeling and falls back to an in-memory MongoDB server when a local MongoDB instance is not configured. Key features implemented include user authentication with email OTP verification, Google Sign-In (demo + real-token verification), product catalog, add-to-cart flow, and order checkout.

Technology Stack

  • Frontend: React 19 + Vite
  • Backend: Node.js + Express
  • Database: MongoDB via mongoose — uses mongodb-memory-server fallback for local runs
  • Auth: JWT tokens, OTP via email (nodemailer), Google ID token verification
  • Styling: Custom CSS with components

Database Details

The project uses mongoose to interact with MongoDB. If MONGO_URI is not set in backend/.env, the application will start an in-memory MongoDB instance using mongodb-memory-server (convenient for local development and testing).

Models:
- User: name, email, password (hashed), role, isVerified, avatarUrl, address, phone, etc.
- Product: name, price, discount, category, description, imageUrl, inventory, seller reference
- Order: customer reference, orderItems (product refs), shippingAddress, paymentMethod, totals
- Otp: email, otp

How to Run Locally

  1. Install dependencies and start backend (from repo root): 
    cd backend
npm install
npm run dev
  2. Start frontend (in another terminal): 
    cd frontend
npm install
npm run dev
  3. Open the app: http://localhost:3000/
To enable real Google Sign-In set GOOGLE_CLIENT_ID in backend/.env. To send real OTP email set GMAIL_USER and GMAIL_APP_PASSWORD.
